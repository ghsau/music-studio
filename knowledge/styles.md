# 风格画像库

> 工作室的**顶层风格决策对象**。每首歌在 `/new-song` 第一步由老周选定一个主画像（可选副画像），写入 `00-brief.md` 的「画像」字段。所有下游角色（观山/墨九/阿声/青衫/抖叔/小汽）按画像执行。
>
> **个人验证归因**（哪首歌验证了哪个画像、哪天升级 / 降级 / 备注）记录在 `notes/styles-attribution.md`，不入 git。本文件只装通用参数 + 规则。
>
> **2026-05-14 加：双模式区分**——画像在两种模式下都用，但参数解释不同：
> - **模式 A 个人 IP**：画像是题材气质对位的工具（按题材选画像）
> - **模式 B 商业爆款**：画像是反向工程爆款的工具（按拆解的爆款结构选画像）
> 详 `knowledge/song-evaluation-framework.md` 双模式段 + `memory/feedback_dual_mode.md`。

## 字段说明

每个画像 8 个字段（+ 状态标记），覆盖整个调度链：

| 字段 | 谁用 | 决定什么 |
|---|---|---|
| 押韵策略 | 墨九 | 写词时押严 / 半押 / 放弃 |
| BPM 区间 | 阿声 | Style prompt 的 BPM |
| 调性偏好 | 阿声 | Key 字段 |
| 意象类型 + 库引用 | 观山 + 墨九 | 读哪本意象库 |
| 人声 | 阿声 | 性别 + 气质 |
| 签名声音 / Suno Persona | 阿声 | 该画像锁定的 Suno Persona（跨歌一致的"高爽的声音"）+ 目标音色定位；详见 `knowledge/vocal-identity.md` |
| 编曲 | 阿声 | 主导乐器 + 节奏感 |
| 情绪光谱 | 观山 + 青衫 | brief 调性 + 视觉调子 |
| 抖音 hook 模式 | 抖叔 + 小汽 | 切片 + 评论钩子选哪类模板 |

## 新增画像的流程

### 决策树：新增 vs 副画像

**该新增**（标 🧪）：
- ≥ 4 个字段（押韵 / 意象 / 编曲 / 人声 / BPM）整体不同
- 至少能预见跑 ≥ 2 首歌的方向
- 现有画像组合无法表达（强行组合反而扭曲）

**不该新增**（用现有画像或副画像）：
- ≤ 3 个字段不同 → 现有画像内的子变体
- 一次性试验，不会重复
- 现有画像 + 副画像可以组合表达

> **核心防膨胀**：能组合的不新增。

### 触发新增的两条路径

| 路径 | 谁触发 | 时机 |
|---|---|---|
| **A. 用户驱动** | 用户提"想试 X 风格" | 老周判断现有不够 → 加新画像 🧪 → 跑首歌 |
| **B. 算子驱动** | 算子在 T+7 复盘时识别有效组合 | 复盘后加新画像 🧪 → 提示用户下次可选 |

### 新画像字段填法

1. 用户/老周给**灵感锚点**——一首参考歌 + 一种气质
2. 老周从灵感倒推 8 字段
3. 字段填不准的位置写 `[待实测]`，跑首歌后回填
4. 整体标 🧪
5. **「签名声音 / Suno Persona」字段必填目标音色定位**（Persona 名/ID 留 `[待回填]`，主理人首跑后在 Suno 创建并回填）；目标音色定位参 `knowledge/vocal-identity.md` 板块 1 选词

## 升级 🧪 → ✅ 的标准

**被 ≥2 首歌经 T+7 复盘 + 算子数据归因验证**。

| 阶段 | 状态 | 行为 |
|---|---|---|
| 第 1 首跑某 🧪 画像 | 保持 🧪 | 即使效果好也只是单一案例，不升 |
| 第 2 首再用此画像 | T+7 复盘时由算子升级 → ✅ | 两次都不翻车（无大 bug + 无明显违和）+ 数据支持 |
| ✅ 后 | 题材匹配时老周倾向采纳 | 不强制百分比；mode 是 advisory（参 老周 §7） |

⚠️ **不可凭"用过 N 首"直接打 ✅**——必须经数据归因。**归因必须追源到具体视频/动作**（看 `memory/feedback_review_attribute_root_cause.md`），不能只看汇总数据。

## ✅ 状态生命周期

✅ 是**阶段性结论**，不是永久 stamp。每次升级 ✅ 时算子加：

```
- verified_at: YYYY-MM-DD                  # 升级日期
- verified_by: 算子复盘 + <曲名 1, 曲名 2>  # 来源数据（写到 notes/styles-attribution.md）
- reverify_due: <verified_at + 90d>        # 重核到期日
```

### 自动降级触发（✅ → 🧪）
1. **reverify_due 到期未再被验证**：老周扫到自动降回 🧪（待重新验证）
2. **算子复盘发现该画像新作品数据明显下降**：直接降 🧪
3. **风格 / 听众口味 / 平台算法变化导致老 ✅ 不再有效**：算子主动降级 + 注明原因

### 降级 ≠ 失败
画像降回 🧪 不是被否定，只是"上次结论需重新验证"。再跑出 ≥ 2 首验证后可重新升 ✅。

### 算子复盘双向能力
- 🧪 → ✅：被 ≥2 首歌经数据归因验证（升级标准）
- ✅ → 🧪：reverify_due 到期 / 数据下降 / 外部环境变化

**核心原则**：今天 ok 的不一定永远 ok；今天不行的未来可能行。**动态看，不固化**。

## 防膨胀硬约束

1. **画像总数上限：8 个**
   - 超过 8 个 → 算子自动触发"画像合并 review"，把相似画像合并
   - 8 个够覆盖中文流行音乐主流细分（国风慢/快/电/戏 + 民谣 + 流行 + 摇滚 + 自由叙事）

2. **🧪 状态有效期：90 天**
   - 90 天内未被第 2 首歌验证 → 算子建议归档（移到"实验中-停滞"区，不删但不再优先选用）
   - 防止占着位置不出活的实验画像
