>
> 本文件是 `knowledge/rap.md` 的配套素材库。**押韵的「方法」（单押/双押/多押怎么判定怎么构造、flow 怎么排）看 `rap.md` 第三节；挑「韵脚素材」（收字、双押词组）查本文件。**

## 一、韵区框架：十三辙

中文说唱押韵的权威分类是**十三辙**（北方说唱艺术的传统韵母归类，按韵腹相同/相近归并）。本库按十三辙组织，每辙给：① 常用收尾单字表；② 双押词组库（核心）。

辙名与韵母对照（速查）：

| 辙名 | 涵盖韵母 | 工作室简称 |
|---|---|---|
| 江阳辙 | ang / iang / uang | ang 区 |
| 言前辙 | an / ian / uan / üan | an 区 |
| 中东辙 | eng / ing / ong / iong | eng/ong 区 |
| 人辰辙 | en / in / un / ün | en 区 |
| 遥条辙 | ao / iao | ao 区 |
| 由求辙 | ou / iou(iu) | ou 区 |
| 一七辙 | i / ü / er | i 区 |
| 姑苏辙 | u | u 区 |
| 灰堆辙 | ei / uei(ui) | ei 区 |
| 怀来辙 | ai / uai | ai 区 |
| 发花辙 | a / ia / ua | a 区 |
| 梭波辙 | e / o / uo | e/o 区 |
| 乜斜辙 | ê / ie / üe | ie 区 |

**判定纪律（再强调，治误报）**：双押 = 末尾**连续两字**韵母逐字都连，且声调标准上一致。挑词时从词尾往前逐字比，第 2 个字一断就是单押。本库「双押词组」栏每条都已标连韵的是哪两字 + 声调，可直接信用。

## 二、用法（墨九）

写 `chinese-rap` 歌、定整首韵脚时：
1. 先按情绪选韵区（每个韵区标了情感倾向，呼应 `guofeng-rhyme.md` 的「副歌 hook 常用韵脚」观察）。
2. 主力押法是双押 → 直接从该韵区「双押词组库」挑现成词组；每条已标连韵两字 + 声调，挑声调一致的优先（Suno 不会主动覆盖声调，见 `rap.md` 第五节）。
3. 单字表用于补单押韵脚、或自己拼新双押。
4. 库里没有合适的 → 现造，但**必须按 `rap.md` 第三节判定法逐字验证**，别误报。新造好用的回填进来。

## 三、可累积机制（核心）

**这个库是活的，越用越厚。** 仿算子复盘 dedup 回填的模式：

- **谁回填**：墨九。**什么时候**：每写完一首 `chinese-rap` 歌的 `01-lyrics.md` 后，作为收尾动作（呼应 `rap.md` 第七节「写 rap 的最小动作清单」）。
- **回填什么**：这首歌里**新用到、且自检 / 试唱确认押得顺**的——① 本库没有的双押词组；② 本库没有的好用收尾单字。
- **回填到哪一节**：默认进对应韵区的「实验中 🧪」种子库，按韵区追加。格式与现有条目一致（双押必标连韵两字 + 声调）。
- **dedup**：回填前扫该韵区，已有的不重复加。
- **升级到 ✅**：墨九不自己升级。算子在 T+7 复盘时，对**经 ≥2 首歌实唱验证、确认 Suno 唱出来押得响不被旋律抹平**的双押组合 / 韵脚，移到「已验证 ✅」节，带 `verified_at` + `reverify_due`（+90d）。
- **降级**：✅ 条目到期未重核、或某首歌复盘发现该韵在 Suno 实唱里押不响 → 算子降回 🧪。

库满了不是问题——韵库就是要厚。但单字表若某韵区超过约 40 字、双押库超过约 30 条，墨九回填时顺手把明显冷僻、从未被实唱选用的旧条目清掉，保持「常用、好用」。
